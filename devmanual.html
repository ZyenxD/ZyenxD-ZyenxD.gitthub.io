Manual de desarrollo

  • Requerimientos iniciales.

    • El Proyecto consiste en la detección de caras y emociones dentro de un archive de video. Los requerimientos que implican los mismo consisten en:

    • Identificar si o no hay un rostro en una imagen
    • Identificar la emoción de esta

  • Análisis del sistema.

    • El Sistema consiste en una aplicación Web, la cual le permita al usuario identificar sus emociones y la presencia de un rostro utilizando la cámara de video de uno de sus dispositivos personales. Esta información es obtenida a través de un servicio web el cual recibe imágenes o fragmentos de las mismas y predice la emoción de dichas basándose en el entrenamiento recibido.

  • Diagrama en bloques de la constitución interna (Entradas/Procesos/Validaciones/Salidas).

  • Diagramas de flujo de la lógica de funcionamiento.

  • Descripción del o los algoritmos o modelos empleados. Justificación de su uso.

    • Para este caso utilizamos RELU (Rectified Linear Unit), ya que presentaba un mayor rendimiento en el entrenamiento del modelo durante las pruebas.

  • Detalles de las Estructuras de Datos/Objetos/Archivos Planos/Bases de Datos empleadas.

    • Para este Proyecto se utilice la data de pruebas de Fer2013.csv, cual contiene una gran cantidad de fotos de expresiones faciales. Estas imágenes contienen las dimensiones de 48*48 en pixeles y esta divididas en pruebas y entrenamiento. Además, estas esta dividida de forma categórica en 7 emociones:

    • tristeza
    • Felicidad
    • enojo
    • neutralidad
    • disgusto
    • miedo

  • Justificación de los datos de prueba empleados.

    • Esta base de datos es una de las mas usadas para este tipo de proyecto por su gran variedad de imágenes y emociones

  • Análisis de los resultados de las pruebas realizadas.

    • Los resultados que se obtuvieron durante el entrenamiento de la data fueron de un 0.85 de accuracy. Lo cual nos indica que las predicciones hechas con el modelo en si, serán bastante aciertas
